What To Build
1. Parameter Sweep Runner
New file: src/backtest/sweep.js
A script that runs the existing backtestDirectional() function across a grid of parameter combinations, saves every result to the backtest_results table, and outputs a sorted summary.

Total combinations: 4 time limits × 5 take profits × 4 stop losses × 4 confidence thresholds = 320 per pair. Two pairs = 640 total backtests.
Run time estimate: Each backtest processes ~2,100–4,300 historical data points through the signal engine. At ~10–50ms per backtest, the full sweep should complete in under 1 minute.
Output Format
Every combination gets a row in backtest_results with the params stored in the JSONB params column. After all runs complete, print a summary table sorted by Sharpe ratio (descending), showing the top 20 results for each pair. Include: pair, time_limit, take_profit, stop_loss, confidence_threshold, trades, hit_rate, avg_pnl_bps, sharpe, max_drawdown, profit_factor, and whether it passes graduation.
CLI Interface
npm run sweep  —  runs the full grid for both pairs
npm run sweep -- --pair=ETH/USDC  —  single pair
npm run sweep -- --top=50  —  show top 50 results instead of 20
